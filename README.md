本專案為一個以 Unity 引擎開發的三維撞球模擬系統，結合物理運算、玩家控制、蓄力機制與延伸互動設計，旨在展示基礎遊戲開發流程與多模組整合能力。

---

## 專案概述

本專案建構於 Unity 2022 開發環境，透過 Rigidbody 物理系統模擬球體之間的碰撞與運動行為，並以撞球桌作為主要場景，提供使用者一個可操作且具互動性的虛擬物理空間。

除了基本的撞球互動外，專案亦整合了蓄力控制與射擊系統，使玩家能透過不同方式影響場上物件，藉此探索多樣化的遊戲操作可能性。

## 核心功能

### 1. 物理模擬系統

專案核心建立於 Unity 的 Rigidbody 與 Collider 架構之上，實現球體之間的碰撞反應與動態運動。透過物理引擎的即時運算，使球體移動具備自然的速度變化與方向轉換，模擬真實撞球行為。

---

### 2. 玩家控制系統

玩家可透過輸入裝置控制視角和球桿，並對場上物件(球)進行操作。系統內包含基本移動控制、方向調整以及動作觸發(球桿擊球)，使使用者能在三維空間中進行精確操作。

---

### 3. 蓄力機制

本專案實作蓄力控制系統，玩家可透過持續輸入累積能量，並於釋放時轉換為作用力。此機制主要應用於擊球行為，基於正常物理反饋影響球體移動速度與距離，增加操作上的策略性與控制深度。

### 4. 射擊互動系統

專案中包含一套物件生成與發射機制，玩家可觸發子彈物件（Prefab）生成並向前發射。該系統主要用於展示 Instantiate 與物理交互的整合應用，並作為擴展遊戲互動形式的基礎。

### 5. 相機跟隨系統

相機會依附於玩家物件進行平滑跟隨，確保遊戲過程中視角穩定且易於觀察場景變化，提升整體操作體驗。

---

### 6. 分數與介面系統

畫面中提供基本的 UI 顯示，包括分數計數與操作回饋。此系統用於即時呈現玩家行為結果，並作為未來擴充完整遊戲規則的基礎。

---

## 遊戲流程

1. 進入撞球桌場景並初始化物件配置
2. 玩家透過輸入裝置調整視角與操作方向
3. 使用蓄力機制進行擊球或互動
4. 球體依物理規則進行碰撞與移動
5. 系統根據互動結果更新分數或狀態

## 技術架構

* 開發引擎：Unity 2022.3.x
* 程式語言：C#
* 物理系統：Rigidbody / Collider
* 主要模組：

  * Player Controller（玩家控制）
  * Camera Controller（相機系統）
  * Game Data Manager（資料管理）
  * Interaction System（互動系統）
  * UI System（介面顯示）

## 設計理念

本專案以「物理互動」為核心，透過簡化的撞球場景作為載體，逐步整合不同系統模組，包括玩家操作、物理回饋與視覺呈現。透過加入蓄力與射擊機制，嘗試突破單一玩法限制，探索多元互動方式在同一場景中的整合可能性。
